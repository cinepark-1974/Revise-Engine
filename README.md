## 🆕 v3.3.7 — A33 강화 + Wiring 복원 (2026-05-05)

### 핵심 변경

「오랜만에」 R2 작업에서 발견된 **A33 누락 결함 5건**의 근본 원인을 두 층위로 해결했습니다.

#### 층위 1 — 룰의 사정거리 부족
기존 A33은 결함 패턴 1·2·3까지만 다뤘습니다. v3.3.7에서 **결함 패턴 4·5·6·7**을 신설했습니다.

| 패턴 | 신규 룰 | 검출 임계값 |
|------|---------|-------------|
| 4 | 짧은 시그니처 라인 무변주 반복 | 동일 화자 2~7자 라인 3회+ |
| 5 | 씬 간 미러링 다이얼로그 복사 | 두 씬 사이 동일 라인 2개+ 공유 |
| 6 | 모티프 객체·소품 반복 | 핵심 소품 사전 매칭 + 3씬+ 등장 |
| 7 | 답변 라인 글자 단위 복제 | 의문문 직후 5자+ 라인 verbatim 2회+ |

#### 층위 2 — Wiring 누락 (결정적 원인)
v3.3.5에서 만든 `auto_fix_a33_violations` 함수가 **정의만 되고 어디서도 호출되지 않은 채로** 남아 있었습니다. 이게 R2 결함이 그대로 화면까지 빠져나간 결정적 원인입니다. v3.3.7에서 `create_revised_docx` 진입부에 호출 wiring을 복원했습니다.

### Phase 처리 방식

| Phase | 검출 대상 | 처리 |
|-------|----------|------|
| 1 | 같은 씬 ID 중복 (패턴 2) | **자동 제거** |
| 2 | 씬 내 동일 화자 동일 대사 (패턴 3) | **자동 제거** |
| 3 | 작품 전체 8자+ verbatim (패턴 1) | 화면 경고 |
| 4 | 짧은 시그니처 라인 3회+ (패턴 4 ★신규) | 화면 경고 |
| 5 | 씬 간 미러 라인 공유 (패턴 5 ★신규) | 화면 경고 |
| 6 | 모티프 객체 3씬+ (패턴 6 ★신규) | 화면 경고 |
| 7 | 답변 라인 verbatim (패턴 7 ★신규) | 화면 경고 |

자동 수정은 Phase 1·2의 구조적 결함만. Phase 3~7은 **검출·보고만** 하고 변주는 작가 결정 영역으로 남겨둡니다 — Mr. MOON의 "AI 자동 보정 대신 직접 잡는다" 작업 방식과 정렬.
